# Threat model

| Threat                              | Asset and attack path                           | Existing controls                                                                                        | Residual risk, detection, response                                                                                 |
| ----------------------------------- | ----------------------------------------------- | -------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| Cross-tenant access                 | Tenant metadata via forged organisation ID      | Server-derived tenant, membership check, scoped queries, API-key binding, isolation tests, RLS plan      | ORM regression; detect denied-access audit and anomalous IDs, revoke session/key and investigate correlation chain |
| Compromised administrator           | Certificate, CA, role, and SSO configuration    | Step-up, least privilege, approvals, separation of duties, immutable audit                               | Colluding admins; alert privileged changes, suspend principal and roll back configuration                          |
| Stolen API key                      | API and MCP operations                          | One-time display, HMAC hash at rest, scopes, expiry, IP constraints, rotation, revocation                | Valid use before detection; monitor last-use/IP, revoke and replay audit scope                                     |
| Stolen browser session              | Human privileges                                | Secure cookies, rotation, absolute expiry, device visibility, revoke-all, step-up                        | Same-device theft; suspicious-login detection and session revocation                                               |
| SSO misconfiguration                | Authentication routing                          | Test mode, verified domains, signed metadata, break-glass policy, step-up change                         | Organisation lockout; alert changes and activate recovery procedure                                                |
| Malicious connector                 | Secrets, internal networks, certificate targets | Capability interface, encrypted secrets, scope allowlist, worker isolation, SSRF controls, dry-run       | Provider compromise; disable connector, rotate credential, validate affected deployments                           |
| Prompt injection                    | Agent plans and tool calls                      | Untrusted-content labels, fixed system policy, schema tools, allowlists, limits, policy gate, no secrets | Model manipulation within allowed reads; detect abnormal tool sequences and halt run                               |
| Agent overreach                     | Bulk lifecycle state                            | Dry-run default, affected-resource cap, approval checkpoints, idempotency, post-verification             | Bad authorised plan; cancel job, rollback deployment, review audit trail                                           |
| MCP client compromise               | Certificate metadata and allowed actions        | OAuth/API key binding, tool-level scope, short sessions, response limits, audit                          | Metadata exposure in scope; revoke client and export correlated calls                                              |
| Private-key exfiltration            | Private key custody                             | Separate custody boundary, import/output guards, envelope encryption, no MCP export                      | Custody provider compromise; rotate wrapping key and affected certificates                                         |
| Audit tampering                     | Compliance evidence                             | Append-only grants, HMAC chain, archival, verification job                                               | Database superuser; compare archived signed batches and initiate incident process                                  |
| SSRF / scan abuse                   | Internal network and cloud metadata             | Approved scopes, CIDRs, protocol/port limits, rebinding and metadata blocks, rate/time limits            | Misapproved broad scope; stop scan, revoke connector, review endpoints                                             |
| Certificate mis-issuance / rogue CA | Trust and identity                              | Profiles, issuer allowlist policy, approval, CT checks, CA capability scope                              | CA compromise; emergency revoke/replace and distrust connector                                                     |
| Supply-chain compromise             | Application and build                           | Lockfile, non-root image, dependency audit, SBOM/signing release plan                                    | Malicious allowed dependency; isolate deployment, rotate secrets, rebuild known-good                               |
| Webhook replay                      | Downstream automation                           | Event ID, timestamp, HMAC signature, secret rotation, delivery log                                       | Consumer ignores replay window; disable endpoint and rotate secret                                                 |
| Renewal race                        | Certificate deployments                         | Fingerprint uniqueness, idempotency, advisory locks, durable state machine, optimistic version           | External non-idempotent target; rollback and reconcile connector state                                             |
| Revocation mistake                  | Availability and trust                          | Step-up, purpose, two-person approval for critical assets, dry-run evidence, CA confirmation             | Approved human error; replacement workflow and incident linkage                                                    |
